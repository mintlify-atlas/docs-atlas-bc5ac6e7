---
title: "Logging API"
description: "API reference for TDLib's logging configuration methods."
---

TDLib exposes logging control through synchronous API requests and a C callback. All logging methods can be called before creating any TDLib client.

## Synchronous requests

These requests are handled by `td_execute` / `ClientManager::execute` — do not use `td_send`.

### `setLogVerbosityLevel`

Sets the verbosity level of TDLib's internal log.

<ParamField path="new_verbosity_level" type="int32" required>
  The verbosity level. `0` = fatal only, `1` = errors, `2` = warnings, `3` = info, `4` = debug, `5` = verbose (default). Values `6`–`1024` enable increasingly detailed trace output.
</ParamField>

```python
td_execute(json.dumps({
    "@type": "setLogVerbosityLevel",
    "new_verbosity_level": 1
}))
```

---

### `getLogVerbosityLevel`

Returns the current log verbosity level.

**Returns:** `logVerbosityLevel` object with field `verbosity_level: int32`.

```python
result = td_execute(json.dumps({"@type": "getLogVerbosityLevel"}))
print(result["verbosity_level"])
```

---

### `setLogStream`

Sets where TDLib writes log output.

<ParamField path="log_stream" type="LogStream" required>
  One of: `logStreamDefault`, `logStreamFile`, or `logStreamEmpty`.
</ParamField>

**`logStreamDefault`** — writes to stderr (or OS log on mobile).

**`logStreamFile`**:

<ParamField path="path" type="string" required>
  Path to the log file.
</ParamField>
<ParamField path="max_file_size" type="int64" required>
  Maximum file size in bytes before auto-rotation. Default: `10485760` (10 MB).
</ParamField>
<ParamField path="redirect_stderr" type="bool">
  If true, also redirect stderr to this file.
</ParamField>

**`logStreamEmpty`** — disables all log output.

```python
# Log to file
td_execute(json.dumps({
    "@type": "setLogStream",
    "log_stream": {
        "@type": "logStreamFile",
        "path": "/var/log/tdlib.log",
        "max_file_size": 10485760,
        "redirect_stderr": False
    }
}))
```

---

### `getLogStream`

Returns the current log stream settings.

**Returns:** A `LogStream` object describing the active stream.

---

### `getLogTags`

Returns a list of TDLib internal log tags that can be used for fine-grained verbosity control.

**Returns:** `logTags` object with field `tags: vector<string>`.

---

### `setLogTagVerbosityLevel`

Sets the verbosity level for a specific internal log tag, overriding the global level for that tag.

<ParamField path="tag" type="string" required>
  A tag name from `getLogTags`.
</ParamField>
<ParamField path="new_verbosity_level" type="int32" required>
  The verbosity level for this tag.
</ParamField>

---

## C callback

### `td_set_log_message_callback`

```c
void td_set_log_message_callback(int max_verbosity_level, td_log_message_callback_ptr callback);
```

Registers a C function to receive log messages directly.

<ParamField path="max_verbosity_level" type="int" required>
  Only messages at or below this level are delivered to the callback.
</ParamField>
<ParamField path="callback" type="function pointer" required>
  `void (*)(int verbosity_level, const char *message)`. Pass `NULL` to remove the callback.
</ParamField>

The C++ equivalent is `ClientManager::set_log_message_callback`.

<Warning>
Never call any TDLib methods from within the log callback. If `verbosity_level == 0` (fatal), TDLib will crash immediately after your callback returns.
</Warning>
