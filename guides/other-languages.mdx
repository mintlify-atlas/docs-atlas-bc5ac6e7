TDLib works from any language that can call C functions. The [JSON interface](/guides/json-interface) is the universal bridge: load `libtdjson`, call the four C functions, and exchange JSON strings. Dozens of community libraries wrap this interface for specific languages.

## Major language bindings

<CardGroup cols={2}>
  <Card title="Go" icon="code" href="https://github.com/zelenin/go-tdlib">
    **go-tdlib** — TDLib client for Go using Cgo and the JSON interface. Supports static and dynamic linking, and includes a generator for TDLib API classes.
  </Card>
  <Card title="Rust" icon="code" href="https://github.com/antonio-antuan/rust-tdlib">
    **rust-tdlib** — Async Rust client with automatically generated, fully-documented types for all TDLib API methods and objects.
  </Card>
  <Card title="Swift" icon="code" href="https://github.com/Swiftgram/TDLibKit">
    **TDLibKit** — Swift package with automatically generated and fully-documented classes for all TDLib API methods. Supports iOS, macOS, watchOS, tvOS, and visionOS.
  </Card>
  <Card title="JavaScript / Node.js" icon="code" href="https://github.com/eilvelia/tdl">
    **tdl** — Fully-asynchronous Node.js interface for TDLib with many examples. Also see **Airgram** for a TypeScript/JavaScript framework.
  </Card>
  <Card title="Python" icon="code" href="https://github.com/alexander-akhmetov/python-telegram">
    **python-telegram** — asyncio-based Python wrapper with full documentation and pip/Docker support. Also see **aiotdlib** and **Pytdbot** for Python 3.9+.
  </Card>
  <Card title="PHP" icon="code" href="https://github.com/aurimasniekis/php-ffi-tdlib">
    **ffi-tdlib** — PHP 7.4+ wrapper using the PHP FFI extension to call TDLib directly without a compiled PHP extension.
  </Card>
  <Card title="Ruby" icon="code" href="https://github.com/southbridgeio/tdlib-ruby">
    **tdlib-ruby** — Ruby bindings and a high-level client for TDLib using the JSON interface.
  </Card>
  <Card title="Dart / Flutter" icon="code" href="https://github.com/ivk1800/tdlib-dart">
    **tdlib-dart** — Dart client with automatically generated and fully-documented classes for all TDLib API methods. Works with Flutter via Dart FFI.
  </Card>
</CardGroup>

## Any language without a listed binding

If your language is not listed, you have two options:

1. **Call the JSON interface directly.** Any language with a C FFI and a JSON library can call `td_create_client_id`, `td_send`, `td_receive`, and `td_execute`. See the [JSON interface guide](/guides/json-interface) for the complete pattern.

2. **Use a CLI bridge.** [tdbot](https://github.com/vysheng/tdbot) and [TDLib JSON CLI](https://github.com/oott123/tdlib-json-cli) provide a command-line tool that exposes TDLib over stdin/stdout as JSON, so any language that can spawn a process and read its output can talk to TDLib.

<Tip>
  If you create a new TDLib binding, consider opening an issue in the [tdlib/td](https://github.com/tdlib/td) repository to have it added to the official examples list.
</Tip>
