TDLib (Telegram Database Library) is an open-source, cross-platform library that provides everything you need to build a fully-functional Telegram client. It handles all network communication, encryption, local storage, and update delivery — so you can focus on building your application.

## Why TDLib?

<CardGroup cols={3}>
  <Card title="Cross-Platform" icon="layer-group">
    Runs on Android, iOS, Windows, macOS, Linux, WebAssembly, and more. One library for all platforms.
  </Card>
  <Card title="Any Language" icon="globe">
    A universal JSON/C interface means you can use TDLib from Python, Go, Ruby, Rust, or any language that can call C functions.
  </Card>
  <Card title="Fully Async" icon="bolt">
    Requests never block each other. Send thousands of concurrent requests and receive responses as they arrive.
  </Card>
  <Card title="Secure by Default" icon="shield">
    All local data is encrypted with a user-provided key. MTProto encryption is handled automatically.
  </Card>
  <Card title="Reliable" icon="signal">
    TDLib remains stable on slow or unreliable connections, with automatic reconnection and consistent update ordering.
  </Card>
  <Card title="Well Documented" icon="file-lines">
    Every API method and class in the TL schema is fully documented with types, descriptions, and usage notes.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Build TDLib">
    Compile TDLib for your target platform. Follow the [manual build guide](/building).
  </Step>
  <Step title="Choose your interface">
    Pick the interface that works best for your language: [JSON](/guides/json-interface) (works everywhere), [C++](/guides/cpp-interface), [Java/JNI](/guides/java-interface), or [.NET](/guides/dotnet-interface).
  </Step>
  <Step title="Authenticate">
    Follow the [authorization flow](/concepts/authorization) to connect to Telegram with your `api_id` and `api_hash`.
  </Step>
  <Step title="Send your first request">
    Load chats, send messages, and handle real-time updates. See the [quickstart](/quickstart) for a working example.
  </Step>
</Steps>

<Note>
  You must obtain your own `api_id` and `api_hash` from [my.telegram.org](https://my.telegram.org) before making any requests.
</Note>
